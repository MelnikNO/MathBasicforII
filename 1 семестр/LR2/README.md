# Отчет по лабораторной работе №2
## Классификация вин с помощью нейронной сети

## Цель работы

Достичь точности классификации на тестовой выборке **выше 94%** с использованием нейронной сети для классификации вин по их химическим характеристикам.

**Задачи:**
- Загрузить и подготовить данные
- Построить нейронную сеть с использованием Keras
- Провести серию экспериментов для достижения целевой точности
- Зафиксировать количество параметров модели

---

## Загрузка и подготовка данных

Для выполнения работы был использован датасет `load_wine()` из библиотеки `sklearn.datasets`. Датасет содержит **178 образцов** вина, каждый из которых описан **13 признаками**. Целевая переменная представляет собой **3 класса** вин.

### Этапы подготовки данных:

1. **One-Hot Encoding**  
   Целевая переменная (`y_data`) была преобразована в формат one-hot encoding с помощью `utils.to_categorical`.

2. **Разделение на выборки**  
   Данные были разделены на три выборки в следующем соотношении (по шаблону):
   - Обучающая выборка: **144 образца**
   - Валидационная выборка: **16 образцов**
   - Тестовая выборка: **18 образцов**

3. **Стандартизация признаков**  
   Для ускорения сходимости модели и улучшения ее работы все признаки были стандартизированы с помощью `StandardScaler`. Масштабирование проводилось на основе обучающей выборки, а затем применено к валидационной и тестовой выборкам.

## Выводы

### Основные результаты

- **Цель достигнута:** точность на тестовой выборке составила **94.44%** (> 94%)
- Количество параметров финальной модели: **13,603**
- Модель успешно классифицирует все 3 класса вин

### Ключевые факторы успеха

1. **Нормализация данных** (StandardScaler) значительно ускорила сходимость модели
2. **BatchNormalization** стабилизировал обучение и позволил использовать более высокий learning rate
3. **Dropout** эффективно предотвратил переобучение на малом датасете
4. **Глубокая архитектура** (4 скрытых слоя) позволила изучить сложные зависимости в данных

### Трудности и решения

| Проблема | Решение |
|----------|---------|
| Малый размер датасета (178 образцов) | Использование регуляризации (Dropout) |
| Риск переобучения | BatchNormalization + Dropout |
| Нестабильная сходимость | Оптимизатор Adam + стандартизация признаков |

### Рекомендации по улучшению

1. Попробовать другие оптимизаторы (SGD с моментом, RMSprop)
2. Экспериментировать с learning rate (learning rate scheduling)
3. Использовать Early Stopping для предотвращения переобучения
4. Добавить больше слоев или нейронов (если данные позволяют)
5. Попробовать L1/L2 регуляризацию вместо Dropout
